Fraud Detection

Detection of Fraud using interpretable methods

Proposal for Artificial Intelligence (AI) in Finance Global Challenge (Problem statement #10)

How can AI automatically flag potential risks and fraudulent activities without human intervention, while maintaining the required levels of customer privacy?

Above shows the overview of the proposed solution for fraud detection.

In our proposed solution, we will train an explanable boosting machine (EBM) and Predict fraud using Large language model through referencing from a generalised dataset.

This dataset can be made up of structured and unstructured data. Structured data can be feed into the models for training directly. Unstructured data will be converted to contextual embedding before feeding into the EBM model.

Both outputs from the EBM and LLM are interpretable to enable the user to understand how the models made the prediction.

The outputs of the EBM and LLM models will be weighted differently to classify the fraud warning as high or low.

Cases that are predicted to have high level of fraud warnings will be prioritize for investigation first. Hence, optimizing the allocation of manpower for investigation.

Details of LLM for fraud detection

Above shows the details of using LLM to detect fraud

A generalised set of data will be save as contextual embedding in the VectorStore for reference.

When the user make a query for a fraud prediction. The LLM will make reference from the vectorStore to make prediction for potential fraud.

From actual testing, the LLM is not able to give consistent result for fraud prediction. To address this, we will be using Reinforced learning human feedback and Low-rank adapters to improve its ability to make fraud predictions.

Generalisation of subsampled data

K-means is used to obtain a subsampled data from the raw dataset to ensure good generalisation of a subsampled data. This is because it is very costly to save all the data in the vectorstore for reference whenever a prediction is needed from the LLM.

EBM global explanability

The EBM provides a global explanability for the relationship between the features and the label.

EBM global explanability can help to track concept drift in data and automatically trigger the re-training of the AI models if required.

EBM local explanability

EBM is used to explain the prediction based on the effects of the features on the label prediction.

Prediction from EBM

The reasoning from EBM will then be compared to the LLM for consistency.

The EBM and LLM will form an ensemble model for fraud prediction.

Even without any human intervention, the EBM is highly effective in predicting fraud cases with very high recall and precision.

Advantage of interpretable ensemble model

EBM and LLM works hand in hand to provide a better solution. EBM is able handle structured data better while LLM model is able to handle unstructured data better.
